# סביבת עבודה לפיתוח VBA

## מבנה תיקיות

- **Access-Projects** - פרויקטים של Microsoft Access
- **Excel-Projects** - פרויקטים של Microsoft Excel  
- **VBA-Library** - ספריית קוד ופונקציות שימושיות

## סוגי קבצים נתמכים

- `.bas` - מודולים סטנדרטיים
- `.cls` - מחלקות ומודולים של מחלקה
- `.frm` - טפסים
- `.vba` - קבצי VBA כלליים

## הגדרות עורך

- גודל פונט: 14
- גופן: Consolas
- גודל טאב: 4 רווחים
- עטיפת שורות: מופעלת
- הצגת רווחים לבנים: בבחירה
- ניקוי רווחים מיותרים: אוטומטי

## תוספים מותקנים

- PowerShell - לתסריטים ואוטומציה
- TypeScript - להשלמת קוד מתקדמת
- Tailwind CSS - לעיצוב
- Trailing Spaces - לניקוי רווחים

## דרכי עבודה מומלצות

### 1. עבודה עם Excel
1. פתח את Excel והפעל את עורך VBA (Alt+F11)
2. כתוב ובדוק קוד בעורך VBA של Excel
3. העתק קוד לקבצים בתיקיית Excel-Projects
4. השתמש ב-Windsurf לניהול גרסאות ותיעוד

### 2. עבודה עם Access
1. פתח את Access והפעל את עורך VBA (Alt+F11)
2. פתח מודולים חדשים או קיימים
3. שמור קוד בתיקיית Access-Projects
4. השתמש ב-Windsurf לארגון ותיעוד

### 3. ספריית קוד משותפת
- שמור פונקציות שימושיות בתיקיית VBA-Library
- יצור קבצים לפי קטגוריות (לדוגמה: Database.bas, Utilities.bas)
- השתמש בקוד בפרויקטים שונים

## טיפים לפיתוח VBA

- השתמש ב-`Option Explicit` בתחילת כל מודול
- הוסף הערות מפורטות לפונקציות מורכבות
- בדוק קוד תדיר בסביבת Office
- שמור גרסאות ישנות לפני שינויים גדולים
- השתמש בשמות משתנים ברורים ועקביים

## קיצורי מקלדת שימושיים

- `Alt+F11` - פתיחת עורך VBA
- `F5` - הרצת קוד
- `F9` - הוספת/הסרת נקודת עצירה
- `Ctrl+G` - פתיחת חלון Immediate
- `Ctrl+R` - הצגת חלון Project Explorer
